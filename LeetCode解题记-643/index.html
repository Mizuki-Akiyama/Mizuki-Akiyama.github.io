643. 子数组最大平均数 I

滑动窗口算法的学习

给你一个由 n 个元素组成的整数数组 nums 和一个整数 k 。

请你找出平均数最大且 长度为 k 的连续子数组，并输出该最大平均数。

任何误差小于10^-5的答案都将被视为正确答案。

示例 1：

输入：nums = [1,12,-5,-6,50,3], k = 4
输出：12.75
解释：最大平均数 (12-5-6+50)/4 = 51/4 = 12.75

示例 2：

输入：nums = [5], k = 1
输出：5.00000

看到这道题，我的第一想法是定义两个指针和两个记录变量

int i = 0;
int j = k - 1;
double max = 0;
double count = 0;

在通过两层循环不断计算指针范围内的和的平均数，并与最大值做比较，如果大于最大值则更新最大值，最后返回最大值

于是有了这段代码

class Solution {
    public double findMaxAverage(int[] nums, int k) {
        double max = 0;
        int i = 0;
        int j = k - 1;
        double count = 0;
        while (j != nums.length) {
            for (int l = i; l <= j; l++) {
                count += nums[l];
            }
            max = Math.max(max, count / k);
            i++;
            j++;
            count = 0;
        }

        return max;

    }
}

于是我想到max的初始值应该更小，于是修改了一下初始值

(ps：最开始选择的是Double.MIN_VALUE，结果还是这个错误。后来查了才知道Double.MIN_VALUE是非0非负数)

这个用例算是过了，却是超出时间限制，卡在（122/127）的地方

看了评论说double类型运算会比int类型慢，以为问题出在了这里，来来回回修改了好几个变量的类型，但还是不能通过全部测试用例

看来只能写一个**O(n)**复杂度的解法了

但是根本想不出来(；д；)

求助了一下Qwen2.5，终于也是把这个逻辑弄懂了

并不需要每次都算出滑动窗口内的平均值，只需要先将max初始化为前k个元素的和，再从第k+1个元素开始滑动窗口，每次移动窗口都计算窗口内的和，并与max比较，最后返回max / k即可

具体计算方法是

max = max - nums[i - k] + nums[i];

通俗来讲，nums[i - k]就是从前k个元素的和去掉第一个元素，这样k个元素的和就只剩k-1个元素。再加上当前元素，k-1个元素的和就又回到了k个元素的和

class Solution {
    public double findMaxAverage(int[] nums, int k) {
        int sum = 0;
        for (int l = 0; l < k; l++) {
            sum += nums[l];
        }
        
        int max = sum;
        
        for (int i = k; i < nums.length; i++) {
            sum = sum - nums[i - k] + nums[i];
            max = Math.max(max,sum);
        }

        return 1.0 * max / k ;

    }
}

其实很好理解，但是脑子如果转不过来那个弯就会被卡的死死的动弹不得啊
