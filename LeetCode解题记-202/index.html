202.快乐数

快慢指针的用法

对于这道题，我认为最关键的一点是能不能找到以下三种可能，并且理解为什么第三种情况永远不会发生：

  1. 最终会得到 1。
  2. 最终会进入循环。
  3. 值会越来越大，最后接近无穷大。

根据下表，可以知道对于 3 位数的数字，它不可能大于 243。这意味着它要么被困在 243 以下的循环内，要么跌到 1。4 位或

4 位以上的数字在每一步都会丢失一位，直到降到 3 位为止。所以我们知道，最坏的情况下，算法可能会在 243 以下的所有数字上循环，然后回到它已经到过的一个循环或者回到 1。但它不会无限期地进行下去，所以我们排除第三种选择。

Digits Largest Next
1 9 81
2 99 162
3 999 243
4 9999 324
13 9999999999999 1053

所以这道题的本质其实是判断是否循环的一个问题。

那么就可以用最经典的快慢指针算法，定义一个 slow 指针和一个 fast 指针，fast 每次走两步，slow 每次走一步，如果二者最终相遇，则表示这是一个循环；反之则 fast 必定先到达终点，也就是 1。

开始解题

定义一个 getNext 方法，参数为 int n，返回值为 int。这个方法的目的是找到当前 n 的每个数位的平方和的和。

public static int getNext(int n) {
    int sum = 0;
    while (n > 0) {
        int d = n % 10;
        sum += d * d;
        n = n / 10;
    }
    return sum;
}

然后在 isHappy 方法中定义快慢指针，使用 while 循环以 fast != slow 作为循环条件重复指针前进的步骤。当 fast == 1 的时候停止循环并返回 true，如果循环正常终止则返回 false。

public boolean isHappy(int n) {
    int slow = n;
    int fast = getNext(n);

    while (fast != slow) {
        slow = getNext(slow);
        fast = getNext(getNext(fast));
        if (fast == 1) {
            return true;
        }
    }

    return false;
}

最终代码成功通过测试，击败了 100% 的用户。
